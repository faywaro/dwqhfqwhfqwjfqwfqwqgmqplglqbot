# Бот-уведомитель ГдеБЕНЗ (Пермь) → Telegram 

Опрашивает открытый API карты `gdebenz.ru`, следит за заправками и
присылает вам сообщение в Telegram, как только у станции появляется
топливо.

## 🖥 Самый простой способ — окно настроек (GUI)

Не нужно открывать и редактировать код. Запустите:

```bash
python3 gdebenz_gui.py
```

Откроется окно, где можно:
- вписать токен бота и chat_id,
- выбрать город из списка (или вписать координаты вручную),
- задать избранные заправки (список osm_id через запятую — пусто = следить за всеми в радиусе),
- выбрать, при каких статусах слать уведомления (Есть / Очередь / Мало / Лимит),
- задать интервал проверки (в секундах),
- включить отключение проверки SSL (если у вас VPN с собственной SSL-инспекцией — см. раздел ниже),
- нажать **"🧪 Тестовое сообщение"** — проверить, что всё настроено верно,
- нажать **"▶ Запустить мониторинг"** — бот начнёт проверять карту в фоне,
  все действия видно в журнале внизу окна,
- нажать **"■ Остановить"** — когда захотите приостановить.

Настройки сохраняются в `config.json` при нажатии "Сохранить" или "Запустить" —
при следующем открытии окна всё будет уже заполнено.

> Если Python установлен с сайта python.org (Windows/macOS) — `tkinter`
> (библиотека для окна) уже встроена, ничего доустанавливать не нужно.
> На Linux иногда требуется `sudo apt install python3-tk`.

Дальше в этом README — описание тех же настроек и альтернативный запуск
через консоль (`notify_gdebenz.py`) и через GitHub Actions, если понадобится.

## Что нужно сделать (один раз)

### 1. Создать Telegram-бота
1. Откройте Telegram, найдите **@BotFather**.
2. Отправьте `/newbot`, придумайте имя и username (должен заканчиваться на `bot`).
3. BotFather пришлёт **токен** вида `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxx` — сохраните его.

### 2. Узнать свой chat_id
1. Найдите в Telegram бота **@userinfobot**, нажмите Start — он пришлёт ваш числовой ID.
2. Либо: напишите что-нибудь своему новому боту (из шага 1), затем откройте в браузере
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates` и найдите в ответе `"chat":{"id":XXXXXXX`.

### 3. Выбрать избранные заправки (опционально)
Если хотите следить не за всеми АЗС в радиусе, а только за конкретными:
1. На карте gdebenz.ru долго удерживайте нужную метку / откройте её карточку.
2. В DevTools → Network → запрос `nearby` найдите нужную станцию и скопируйте
   её поле `osm_id` (например `"937617949"`).
3. Впишите такие ID в список `FAVORITES` в файле `notify_gdebenz.py`, например:
   ```python
   FAVORITES = ["937617949", "833690595"]
   ```
   Если оставить `FAVORITES = []` — бот будет следить за всеми заправками
   в радиусе 20 км от центра Перми.

### 4. Разместить код на GitHub и включить автозапуск
1. Создайте **новый приватный репозиторий** на github.com (приватный — чтобы
   никто не видел вашу конфигурацию).
2. Загрузите в него содержимое этой папки `gdebenz_bot` целиком, сохранив
   структуру папок (важно: `.github/workflows/gdebenz-watch.yml` должен
   остаться по этому пути). Файл `config.json` из-за `.gitignore` не
   попадёт в репозиторий — это специально, чтобы токен бота не хранился
   в открытом виде в коде; для GitHub Actions токен и chat_id передаются
   через Secrets (шаг 3 ниже).
3. В репозитории откройте **Settings → Secrets and variables → Actions → New repository secret**
   и добавьте два секрета:
   - `TELEGRAM_BOT_TOKEN` — токен из шага 1
   - `TELEGRAM_CHAT_ID` — ваш chat_id из шага 2
4. Откройте вкладку **Actions** в репозитории, разрешите workflow, и он
   начнёт запускаться автоматически каждые 10 минут (это бесплатно для
   обычных публичных и приватных репозиториев в разумных пределах).
5. Чтобы проверить сразу, не дожидаясь расписания — во вкладке Actions
   откройте workflow "GdeBenz Watch" → **Run workflow**.

## Тестовый запуск на своём ПК

Скрипт использует только стандартную библиотеку Python — ничего
дополнительно ставить не нужно, достаточно самого Python 3.

### Шаг 1. Впишите токен и chat_id прямо в файл

Откройте `notify_gdebenz.py`, найдите строки:
```python
TELEGRAM_BOT_TOKEN = os.environ.get("TELEGRAM_BOT_TOKEN", "ВСТАВЬТЕ_ТОКЕН_СЮДА")
TELEGRAM_CHAT_ID = os.environ.get("TELEGRAM_CHAT_ID", "ВСТАВЬТЕ_CHAT_ID_СЮДА")
```
и замените `"ВСТАВЬТЕ_ТОКЕН_СЮДА"` / `"ВСТАВЬТЕ_CHAT_ID_СЮДА"` на реальные
значения (в кавычках). Для локального теста это проще, чем возиться с
переменными окружения.

### Шаг 2. Проверьте, что Telegram настроен верно

```bash
python3 notify_gdebenz.py --test
```
В Telegram должно прийти сообщение "🧪 Тестовое сообщение от бота ГдеБЕНЗ".
Если не пришло — проверьте токен и chat_id (частая ошибка: вы не написали
своему боту "Привет" хотя бы раз — Telegram не разрешает ботам первыми
писать в личку, пока пользователь сам не начал диалог).

### Шаг 3. Разовая проверка карты

```bash
python3 notify_gdebenz.py --once
```
Один раз опросит карту, покажет в консоли сколько станций проверено и
сколько уведомлений отправлено (пришлёт в Telegram, только если у
какой-то станции реально стоит статус из `NOTIFY_STATUSES`).

### Шаг 4. Постоянный локальный запуск (раз в минуту)

```bash
python3 notify_gdebenz.py
```
Без флагов скрипт запускается в бесконечном цикле: проверяет карту,
ждёт `CHECK_INTERVAL_SECONDS` (по умолчанию 60 секунд — раз в минуту),
проверяет снова, и так пока вы не остановите его через **Ctrl+C**.
Компьютер должен быть включён и скрипт запущен всё это время — как только
закроете терминал, проверка остановится.

Интервал можно поменять в файле:
```python
CHECK_INTERVAL_SECONDS = 60
```

> Раз в минуту — нормально для теста. Для постоянной работы советую не
> ставить чаще 3-5 минут, чтобы не создавать лишнюю нагрузку на бесплатный
> сервис gdebenz.ru.

### Ошибка "CERTIFICATE_VERIFY_FAILED: self-signed certificate"

Некоторые VPN делают собственную SSL-инспекцию трафика (подменяют
сертификат на свой). Есть два способа исправить:

- **Правильный:** добавить корневой сертификат вашего VPN в доверенные
  сертификаты системы (в настройках VPN-клиента обычно есть пункт вроде
  "Export root certificate" / "Install CA certificate").
- **Быстрый, для локального теста:** в `notify_gdebenz.py` поменять
  ```python
  DISABLE_SSL_VERIFY = False
  ```
  на
  ```python
  DISABLE_SSL_VERIFY = True
  ```
  Это отключает проверку сертификата для запросов бота. Ослабляет защиту
  от подмены соединения — используйте, только если доверяете своей сети/VPN.
  Для GitHub Actions (сервера не в России) эта проблема не возникает, там
  флаг можно оставить `False`.



| Параметр | Что делает |
|---|---|
| `LAT`, `LON`, `RADIUS_KM` | Центр и радиус поиска заправок (сейчас — Пермь, 20 км) |
| `FAVORITES` | Список osm_id заправок для отслеживания. Пусто = все в радиусе |
| `NOTIFY_STATUSES` | При каких статусах слать уведомление (по умолчанию: есть / очередь / мало) |

## Важно

- Это неофициальный инструмент поверх открытого API сайта gdebenz.ru —
  сайт может изменить формат ответа без предупреждения, тогда скрипт
  придётся поправить.
- Не ставьте интервал опроса чаще, чем раз в 5 минут — чтобы не создавать
  лишнюю нагрузку на чужой бесплатный сервис.
- Данные на карте — это субъективные отметки водителей, а не официальная
  информация о наличии топлива.
